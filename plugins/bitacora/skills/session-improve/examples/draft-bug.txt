### Steps to reproduce

1. Log in on Safari (16 or 17 observed).
2. Click "Log out" in the user menu.
3. Observe the UI returns to the login screen.
4. Open DevTools → Network and inspect the `POST /api/session/end` request.

### Expected

- `POST /api/session/end` returns `204`.
- The session cookie is cleared.

### Actual

- `POST /api/session/end` returns `401` (~10% of attempts, Safari only — see QA repro in comment #2).
- The UI shows the login screen as if logout succeeded.
- The session cookie persists in some cases; reloading the page restores the session.

### Environment

- Safari 16 + 17 confirmed (comment #2). Chrome / Firefox not reproducible.
- All tenants observed; no region pattern (comment #4 from oncall@).

### Notes

- Suspected timing race in the session-end handler (oncall@ in comment #4); the `401` shape
  matches a stale-CSRF code path the recent middleware refactor touched.
- QA's reproduction script is attached to comment #2.
- Assumption: the bug is "logout fails silently" (cookie persists, user thinks they logged
  out) rather than "logout returns `401` visibly" — based on comment #2's "looks logged out,
  isn't" framing. Validate before fixing.
